🔐 Auth Flow
1. POST /api/auth/register
   → bcrypt hashes password (12 rounds)
   → stores user in MongoDB
   → returns accessToken (15 min)

2. POST /api/auth/login
   → timing-safe password check
   → issues accessToken (JWT, 15 min)
   → issues refreshToken (7 days)
   → refreshToken stored as HASH in DB

3. Protected route request
   → verifyToken middleware
   → decodes JWT → fetches user from DB
   → checks isActive flag
   → attaches req.user

4. POST /api/auth/refresh
   → verifies refresh JWT signature
   → finds hashed token in DB
   → rotates: old token removed, new pair issued
